class Cachorro:
    """"Uma tentativa simples de modelar um cachorro"""

    def __init__(self, nome, idade):
        """Inicializa os atributos nome e idade"""
        self.nome = nome
        self.idade = idade

    def sentar(self):
        """Simula um cachorro sentando em resposta a um comando"""
        print(self.nome.title() + " está sentando.")

    def rolar(self):
        """Simula um cachorro rolando em resposta a um comando"""
        print(self.nome.title() + " está rolando!!")
  • Comentando sobre o código acima:

Método init (self, nome, idade):¶

  • Conceitualmente, uma função que faz parte de uma classe chama-se método.
  • Tudo o que foi aprendido em função aplica-se aos métodos.
  • Os dois underscores antes e depois servem para diferenciar nomes default de Python com os criados por nós.
  • O método init (self) é um método especial do Python que é executado automaticamente sempre que uma instância é criada baseada na classe Cachorro.
  • O método recebe 3 parâmetros: self, nome e idade:
    • self: é um parâmetro obrigatório na definição de um método e deve estar sempre antes dos demais parâmetros.
      • Quando o Python chama o método init (self) para criar uma instância de Cachorro, a chamada passará o argumento self automaticamente.
      • Toda chamada de método associada a uma classe passa o argumento self.
      • O argumento é uma referência à própria instância automaticamente, permitindo acesso aos atributos e métodos da instância individual da classe.
    • Não é preciso especificá-lo na chamada. É passado automaticamente por Python.

2.3. Conceitos de Herança¶

  • Podemos utilizar as classes criadas, ou se quisermos criar uma classe especializada, por meio dos conceitos da Herança.
  • Quando uma classe herda de outra, ela assume automaticamente os seus atributos e métodos da classe original, isto é Herança.
    • A classe original que passará seus atributos e métodos chama-se classe-pai, classe-mãe, super classe.
    • E a que recebe, é a classe-filha, subclasse. A classe-filha além de receber os atributos e métodos da classe-pai, ela poderá também, definir seus próprios atributos e métodos.
  • Exemplos Super Classe: Atenção: Vamos usar a classe criada anteriormente.
    • Nome da Classe: Carro <---- É a classe mãe, ou super classe.
    • Atributos da super classe: fabricante, modelo, ano, leitura_hodometro.
    • Métodos da super classe: buscar_nome_descricao( ), ler_hodometro( ), atualizar_hodometro( ), incrementar_hodometro( ).
  • Exemplos Subclasse:
    • Nome da Classe: CarroEletrico <---- É a classe filha, ou subclasse.
    • Atributos da subclasse: bateria que é do tipo de uma classe chamada Bateria.
    • Métodos da subclasse: abastecer_tanque(), descricao_bateria().

# na classe mãe.

class CarroEletrico(Carro):
    """Representa aspectos de um carro específicos de veículos elétricos"""

    def __init__(self, fabricante, modelo, ano):
        """Inicializa os atributos da classe-mãe"""
        super().__init__(fabricante, modelo, ano)
        self.bateria = 70  # capacidade da bateria

Observações:¶

Classe Filha: CarroEletrico(Carro)¶

  • O arquivo inicia com a classe-mãe ---> Carro.
  • As classes-filhas devem estar no mesmo arquivo da classe-mãe, após a sua definição.

  • A classe CarroEletrico é definida após a classe Carro, sendo que a classe-filha recebe entre parênteses a classe-mãe.

    Função super( )

    • O método init () aceita as informações necessárias para criar uma instância de Carro > fabricante, modelo, ano.
    • Inserimos uma função chamada super(), a qual é uma função especial que auxilia o Python a criar conexões entre as classes-filhas e a classe-mãe. Ao chamar a função super(), _indica que deve ser chamado o método init () da classe-mãe de CarroEletrico que confere os atributos da classe-mãe.
    • No Pycharm, ele indica do lado esquerdo, com um símbolo circular de que existe uma subclasse CarroEletrico.

4. Conceitos de Modificadores e Encapsulamento¶

Fracamente¶

  • O Python não utiliza o termo private, que é um modificador de acesso e também chamado de modificador de visibilidade.
  • Existe uma convenção:
    • Métodos e atributos fracamente privados são os que possuem um underscore antes do nome. Essa abordagem apenas sinaliza que essas variáveis são privadas e outros programadores não devem utilizá-las em código externos.
    • Nessa situação conseguimos fazer o acesso de fora da classe.

Encapsulamento¶

  • Encapsular é fundamental para que seu sistema seja suscetível a mudanças:
    • não precisamos mudar uma regra de negócio em vários lugares, mas sim em apenas um único lugar, já que essa regra está encapsulada.
    • Quando utilizamos os underscores estamos alertando para não ler diretamente aquele atributo ou mesmo modificá-lo diretamente. Para isso, utilizar os métodos de acesso get e set.
    • Aprofunde o assunto, lendo Prof. Marcolan.

Fortemente¶

  • Métodos e atributos fortemente privados são os que possuem duplo underscore antes do nome. Essa abordagem sinaliza que essas variáveis são privadas. Não é possível acessá-los de forma padrão.
  • Para isso, fazemos o acesso de fora da classe por: nomeInstância._nomeClasse__nomeatributo. (primeiro é usado um underscore antes do nome da classe, e em seguinda, dois underscores antes do nome do atributo).
  • As mesmas regras de acesso aos atributos valem para os métodos.

    • Escreva um método de nome descricao_usuario() que apresente um resumo das informações do usuário.
    • Escreva outro método chamado greet_usuario que mostre uma saudação personalizada ao usuário.
    • Crie diversas instâncias que representem diferentes usuários e chame os dois métodos para cada instância.

  • Exercício4 - Pessoas Atendidas:

    • A partir do Exercício1, acrescente na classe chamada Restaurante, o atributo pessoas_atendidas cujo valor default é 0.
    • Crie um instância chamada restaurante a partir desta classe, apresente o número de cliente atendidos pelo restaurante e, em seguida, mude esse valor e exiba-o novamente.
    • Acrescente um método chamado set_pessoas_atendidas() que permita definir o número de clientes atendidos. Chame este método com um novo número e mostre o valor novamente.
    • Adicione um método chamado incrementar_pessoas_atendidas() que permita incrementar o número de clientes servidos. Chame esse método com qualquer número que você quiser e que represente quantos clientes foram atendidos, por exemplo, em um dia de funcionamento.

  • Exercício5 - Tentativas de Login:

    • A partir do Exercício3, acrescente um atributo tentativas_login à sua classe Usuario.
    • Escreva um método chamado incrementar_tentativas_login() que incremente o valor do atributo tentativas_login.
    • Escreva outro método chamado reset_tentativas_login() que reinicie o valor de tentativas_login.

  • Exercício6 - Tentativas de Login:

    • Crie uma instância de Usuario e chame incrementar_tentativas_login() várias vezes. Exiba o valor de tentativas_login para garantir que ele foi incrementado de forma apropriada e, em seguida, chame o reset_tentativas_login().
    • Exiba tentativas_login novamente para garantir que seu valor foi reiniciado com 0.
